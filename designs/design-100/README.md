# Design 100 - Client Site Workbench

This exploration uses a table-led management screen as the first viewport. It is aimed at agency developers who need to create, save, rename, delete, import, export, and inspect client test sites without first learning the full Playground mental model.

The design uses beginner-readable action cards, but keeps the table as the dominant structure. Save destinations, destructive actions, import-over-current-site warnings, reset/reload behavior, and export results are shown as concrete states inside the wireframe.
