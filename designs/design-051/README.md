# Design 051: Split Site Manager Takeover

This exploration treats Site Manager as the primary Playground product surface. The first viewport opens directly into a support-oriented runtime workspace with Settings, Files, Blueprint, Database, and Logs tabs, while the live WordPress site is reduced to a narrow verification preview.

The left rail handles environment switching, creation sources, saved Playground management, and export actions so support engineers can keep reproduction setup, file edits, and diagnostics in one dense workspace.
